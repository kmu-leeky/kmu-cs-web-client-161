Understanding CSS Concepts

  1. Describe difference between HTML and CSS with respect to the roles they take in a web page? (5 point)
    2. Answer:

    HTML은 집으로 비유하자면 뼈대를 구축하는 기능을 하는 것이고, CSS는 색상이나 효과를 그 집에 적용해 외관을 꾸미는 역할을 한다. 즉, HTML은 문서의 구조를 담당하고 CSS는 문서의 스타일링을 담당한다.

  2. CSS stands for Cascading Style Sheet, what does Cascading means? (5 point)

    3. 하나의 요소에 가중치가 계산되어서 우선순위에 따라 가중적용이 된다. 우선순위가 적용되는 것이 폭포가 떨어지는 것과 비슷해서 cascading이라고 부른다. 예를들어, a:hover를 하면 a태그에 마우스를 올리는 곳만 바뀌는 것처럼 그 우선순위를 확인해 맞는 곳에 효과를 주는것이다.

  3. List three different methods to use CSS with a HTML document and write a brief decription how to use it(5 point)
    4. Answer:

    1. External : css파일을 외부에 생성해서 html에서 link 태그를 이용해 접근한다. 장점은 다른 html 파일에서도 같은 css파일을 이용할 수 있다.

    <link ref="stylesheet" type="text/css" href="style.css">를 이용하여 사용 가능하다.

    2. Internal : css파일을 html의 head 태그의 style 태그안에 입력해 body에서 사용한다. 단점은 다른 html 파일에서는 css 스타일을 이용할 수 없다.

    3. Inline : html의 body부분에 직접 쓰는 것이다. 예를들어, style="color:blue;"를 p 태그안에 넣어주면 p태그 안에서 그 스타일을 이용할 수 있다.

  5. When we want to set font style using CSS, we can set generic family and font family. What are the difference between them?(5 point) Which one is recommended to state as a last option to show in a web-page? (5 point)
    6. Answer:

    font-family는 사용가능한 글꼴들의 집합체를 만들어주는 것이다. 그 안에 family-name(글꼴이름)을 나열하고, font-family의 제일 마지막에 generic-family를 선언한다. family-name에 적은 순서대로 다운로드 된 글씨체가 있는지 확인하는데 어떤 글씨체도 없다면 마지막의 generic-family를 이용한다.

    두번째 질문의 답은 Generic-family 이다. 왜냐하면 다운로드 된 글씨체(family-name도 포함)가 아무것도 없다면 generic family의 글씨체를 이용해야 하기 때문이다.

  6. When we want to control space of an element, we can use padding and margin. What are the difference between them?(5 point)
    7. Answer: margin은 border밖의 공간의 여백을 설정해주는 것이고, padding은 border와 content사이의 여백을 설정해주는 것이다. 쉽게말하면, margin은 외부여백을 설정해주는 것이고, padding은 내부여백을 설정해주는 것이다.
  7. When we set font size using CSS, we can use units of em, %, px. How are they different? (5 point)
    8. Answer:

    1. em : 브라우저의 default 값이다. (1ea : 약 16px, 약 12pt)

    2. px : font size를 절댓값형태로 나타내는 것이다. 이걸 이용하면 반응형 웹에서는 창의 사이즈에따라 font size가 변하지 않는다. 그래서 em이나 %를 이용한다.

    3. % : 기본값으로부터 변경되는 폰트비율이다. 100%가 default이다.

  8. There are four ways to set conditional property for a sequence of elements (parent-child, siblings) - we called it as Combinator in the class. Describe the name of the four combinators with brief descripton and give a short example (10 point)

    9. 예시

    <div> <p> HI </p> <p> My name is</p> <span><p>Koo Min Jun</p></span> </div> <p> Welcome to</p> <p> My homepage</p>

    1. descendant selector

    태그선언을 두개 할때 그 사이에 공백을 두면 첫번째 태그안에 들어있는 두번째 태그에 효과적용이 된다.

    예를들어, div p{}를 선언하면 div내 선언된 p태그는 모두 효과가 적용이 된다. - HI와 My name is와 Koo Min Jun은 효과적용된다.

    2. Child selector(>)

    태그선언을 두개 할때 그 사이에 >를 넣으면 첫번째 태그안에 들어있는 두번째 태그중 두번째 태그가 다른 태그에 둘러싸여있지 않으면 효과적용이 된다. -HI와 My name is만 효과적용된다.

    예를들어, div>p{}를 선언하면 div내 선언된 p태그에는 효과 적용되겠지만, p를 둘러싸고있는 태그가 있다면 그 a태그는 적용이 되지않는다.

    3. Adjacent sibling(+)

    태그선언을 두개 할때 그 사이에 +를 넣으면 첫번째 태그가 끝난 다음의 두번째 태그 하나만만 적용이 된다. - Welcome to만 효과적용된다.

    예를들어, div+p{}를 선언하면 div태그가 끝난 다음 p에만 효과가 적용된다.

    4. General sibling(~)

    태그선언을 두개 할때 그 사이에 ~를 넣으면 첫번째 태그가 끝난 다음의 두번째 태그들 모두에 적용이 된다.- Welcome to와 My homepage에 효과적용된다.

    예를들어, div~p{}를 선언하면 div태그가 끝난 다음 선언되는 모든 p에 효과가 적용된다.
